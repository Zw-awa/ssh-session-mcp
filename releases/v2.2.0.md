# v2.2.0 - Intelligent Command Completion & Safety Modes

A major reliability and safety upgrade. Commands now complete deterministically, dangerous operations are blocked by default, and long-running commands are handled gracefully.

## Highlights

### Deterministic Command Completion
- **Sentinel markers**: Each command gets a unique completion marker (`___MCP_DONE_<id>_<exitcode>___`). No more guessing when a command finishes.
- **Exit code capture**: `ssh-run` now returns the actual exit code of the command.
- **Triple detection**: Sentinel marker (primary) → prompt pattern matching → idle timeout (fallback). Commands complete as fast as possible.
- Disable with `SSH_MCP_USE_MARKER=false` if needed.

### Safety Modes (safe/full)
- **Safe mode** (default): Blocks dangerous commands (`rm -rf`, `mkfs`), interactive programs (`vim`, `htop`), and streaming commands (`tail -f`). Returns helpful suggestions.
- **Full mode**: AI has full control. Only blocks extreme threats (fork bombs, `dd` to disk devices).
- **Browser toggle**: Switch modes directly from the browser terminal UI. Full mode requires confirmation dialog.
- Configure via `SSH_MCP_MODE=safe|full` or `--mode=safe|full`.

### Async Command Tracking
- Long-running commands (apt install, docker build, etc.) automatically transition to async mode.
- Returns a `commandId` immediately — AI polls with `ssh-command-status`.
- Background monitoring continues for up to 5 minutes.

### New Tools
| Tool | Description |
|------|-------------|
| `ssh-command-status` | Poll async command progress and output |
| `ssh-retry` | Retry failed commands with exponential/fixed backoff |

### Structured Output Parsing
- `ssh-run` automatically parses output of common commands into JSON:
  - `git status` → branch, staged, modified, untracked files
  - `git log` → array of {hash, author, date, message}
  - `ls -la` → array of {name, type, permissions, size, owner}
- Available in the `parsed` field of the response.

### Terminal Mode Awareness
- Detects if terminal is in shell, editor (vim/nano), pager (less/more), or password prompt state.
- Safe mode blocks command execution in wrong terminal states.
- `ssh-status` reports current terminal mode.

## Bug Fixes
- Fixed race condition in `waitForCompletion` where output could be missed between initial check and listener registration
- Fixed shell compatibility issue with exit code capture in piped commands
- Fixed password prompt detection for `[sudo] password for user:` format
- Added mutual exclusion for concurrent `ssh-run` calls on the same session
- Stuck running commands now auto-interrupted after 10 minutes

## Configuration

New environment variables:

| Variable | Default | Description |
|----------|---------|-------------|
| `SSH_MCP_MODE` | `safe` | Operation mode: safe or full |
| `SSH_MCP_USE_MARKER` | `true` | Enable sentinel completion markers |

## Installation

```bash
npm install -g ssh-session-mcp@2.2.0
```
